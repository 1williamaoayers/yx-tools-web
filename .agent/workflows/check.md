# 🔍 自检工作流

## 触发条件
用户说 `/check`、`自检`、`检查一下`、`你违规了吗`

## 执行步骤

### 1️⃣ 回顾最近输出

检查本次对话中我的所有输出：
- 回复内容
- 文件修改
- 命令执行

### 2️⃣ 逐项检查

#### 🔤 语言检查
```
□ 所有回复都是中文吗？
□ 标题是中文吗？
□ 没有大段英文吗？
```

#### 📝 记录检查
```
□ notes.md 锚点是最新的吗？
□ TODO.md 状态准确吗？
□ 代码有中文注释吗？
```

#### ✅ 验证检查
```
□ 说"完成"时有真实测试结果吗？
□ 没有用"应该可以"、"待验证"吗？
□ 打勾的任务都有依据吗？
```

#### 📋 流程检查
```
□ 开工时读了 notes.md 和 TODO.md 吗？
□ 开始任务前更新了锚点吗？
□ 每步都更新了锚点吗？
```

### 3️⃣ 汇报结果

使用以下模板：

```
## 🔍 自检报告

### ✅ 合规项
- [列出做对的事]

### ❌ 违规项（如果有）
- 违规: [具体描述]
- 纠正: [已采取/将采取的措施]

### 📊 总结
[X] 项合规，[Y] 项违规
```

### 4️⃣ 立即纠正

如果发现违规：
1. 不要找借口
2. 立即执行纠正措施
3. 完成后告知用户

---

## 🚫 禁止行为

- ❌ 敷衍检查（"都挺好的"）
- ❌ 隐瞒违规
- ❌ 只承认不纠正

## ✅ 必须行为

- ✅ 诚实汇报每一项
- ✅ 违规立即纠正
- ✅ 纠正完成后确认
