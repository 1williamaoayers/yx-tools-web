# 🔄 断线恢复工作流

## 触发条件
用户说 `/resume` 或 "继续" 或 "断哪了"

## 执行步骤

### 1️⃣ 读取锚点 (最重要)
```
读取 notes.md 的 "🔴 当前锚点" 区域
```
**关注字段**：
- 🎯 正在做
- 📍 当前步骤
- 💡 下一步
- 🔗 相关文件

### 2️⃣ 读取任务状态
```
读取 TODO.md 的 "📌 当前任务" 区域
```
**关注**：带 `🚧` 标记的进行中任务

### 3️⃣ 一致性检查
对比锚点和 TODO：
- ✅ 如果一致 → 直接汇报断点
- ⚠️ 如果不一致 → 提醒用户可能有数据丢失

### 4️⃣ 汇报断点
用这个模板向用户汇报：

```
## 🔄 断点恢复报告

**上次在做**: [从锚点读取]
**做到第几步**: [从锚点读取]
**下一步计划**: [从锚点读取]
**相关文件**: [从锚点读取]

---
继续执行吗？还是需要调整方向？
```

### 5️⃣ 等待用户确认
- 用户说"继续" → 从 `下一步` 开始执行
- 用户说"重来" → 回到任务起点
- 用户说其他 → 按新指令执行

---

## 🚫 禁止行为
- ❌ 不读锚点就瞎猜
- ❌ 不汇报就直接开干
- ❌ 忽略一致性检查

## ✅ 必须行为
- ✅ 先读后说
- ✅ 引用具体内容证明读了
- ✅ 让用户确认后再继续
